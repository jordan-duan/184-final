Summary

We are building a voxel-based simulation, where we can simulate a physically-accurate object, such as a ball or ragdoll object, so it interacts with the world naturally as well as with collisions from the player using the JBullet physics engine. On top of this, we will incorporate modern computer graphic shading techniques like Blinn Phong, real time shadow rendering, reflections, and world environment collisions, all in Minecraft!

Problem Description

In voxel worlds like Minecraft, dynamic objects are rare and generally lack realistic physical behavior or rich modern computer graphics shading. Typically, everything is axis-aligned, block-based, and interacts in limited, non-physically accurate, ways. Our project seeks to explore what happens when you add a smooth, round, physically simulated object (like a ball or object ragdoll) into this environmentâ€”one that reacts to player movement and the environment using real physics.

We'll use JBullet to simulate forces, collisions, and responses between the object, the world, and the player. To enhance visual realism, we'll shade the object using techniques from physically-based rendering (PBR), add real-time shadows, and reflect the environment onto its surface with cube mapping. This blend of accurate motion and stylized rendering adds a new dimension to voxel based gameplay.
