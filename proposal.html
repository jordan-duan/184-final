Summary

We are building a voxel-based simulation, where we can simulate a physically-accurate object, such as a ball or ragdoll object, so it interacts with the world naturally as well as with collisions from the player using the JBullet physics engine. On top of this, we will incorporate modern computer graphic shading techniques like Blinn Phong, real time shadow rendering, reflections, and world environment collisions, all in Minecraft!

Problem Description

In voxel worlds like Minecraft, dynamic objects are rare and generally lack realistic physical behavior or rich modern computer graphics shading. Typically, everything is axis-aligned, block-based, and interacts in limited, non-physically accurate, ways. Our project seeks to explore what happens when you add a smooth, round, physically simulated object (like a ball or object ragdoll) into this environmentâ€”one that reacts to player movement and the environment using real physics.

We'll use JBullet to simulate forces, collisions, and responses between the object, the world, and the player. To enhance visual realism, we'll shade the object using techniques from physically-based rendering (PBR), add real-time shadows, and reflect the environment onto its surface with cube mapping. This blend of accurate motion and stylized rendering adds a new dimension to voxel based gameplay.

Goals and Deliverables

(1) What We Plan to Deliver (Baseline)

  1. Our project should allow us to simulate a rigid object, such as a ball, with accurate collision detection and physics using JBullet.
  2. Player interactions and collisions, such as pushing and hitting, with our rigid objects will be implemented via ray-casting and bounding volume detection.
  3. We plan to implement directional lighting and PBR-style shading, such as diffuse/specular, roughness, and metalness.
  4. We will add real-time shadows to enhance depth and realism.
  5. Finally, we want to render the object in a voxel scene, even if the object itself is represented as a sphere or capsule.

(2) What We Hope to Deliver (Stretch Goals)

  1. Instead of only using spheres, we hope to replace the ball with an articulated ragdoll using joint constraints.
  2. To create more realistic lighting, we can use cube mapping for dynamic environment reflections on shiny surfaces.
  3. Minecraft is a game with many different items and environments! As such, we want to add a material system to shift between soft (cloth-like) and hard (metallic) shading.
  4. We are interested in rendering Minecraft in a unique style by implementing a stylized rendering style, such as toon or outline shaders and pixelation or reverse pixelation, for a more artistic effect.
  5. Finally, to continue to make our physics simulation more accurate, we hope to add soft-body or deformable physics.
